Namespace Buttplug.Core.Messages

Classes

ButtplugDeviceMessage

Subclass of Buttplug Messages, that command a device to take an action.

ButtplugMessage

Base class for Buttplug protocol messages.

ButtplugMessageMetadata

DeviceFeature

Represents a single feature of a device in V4 protocol.

DeviceFeatureInput

Input feature attributes for a device feature.

DeviceFeatureOutput

Output feature attributes for a device feature.

DeviceInfo

Container class for describing a device in V4 protocol.

DeviceList

List of devices connected to the server.

DeviceListConverter

Custom JSON converter for DeviceList that handles the dictionary keyed by device index strings.

Error

Indicator that there has been an error in the system, either due to the last message/query sent, or due to an internal error.

InputCmd

Sent to server, requests a sensor reading or manages sensor subscriptions. Replaces SensorReadCmd and SensorSubscribeCmd from V3.

InputReading

Sent from server, contains a sensor reading from a device. Replaces SensorReading from V3.

InputReadingConverter

Custom JSON converter for InputReading to handle the dynamic Reading property.

InputReadingValue

Value container for sensor readings.

MessageReceivedEventArgs

Event fired when a new ButtplugMessage is received.

Ok

Signifies the success of the last message/query.

OutputCmd

Sent to server, generic message that controls device outputs (vibration, rotation, position, etc.). Replaces ScalarCmd, RotateCmd, and LinearCmd from V3.

OutputCmdConverter

Custom JSON converter for OutputCmd to handle the dynamic Command property.

OutputCommandValue

Output command value for non-duration outputs.

OutputCommandValueWithDuration

Output command value for PositionWithDuration outputs.

Ping

Sent to server, at an interval specified by the server. If ping is not received in a timely manner, devices are stopped and client/server connection is severed.

RequestDeviceList

Sent to server to request a list of all connected devices.

RequestServerInfo

Sent to server to set up client information, including client name and protocol version. Denotes the beginning of a connection handshake.

ScanningFinished

Sent from server when scanning has finished.

ServerInfo

Sent from server, in response to RequestServerInfo. Contains server name, protocol version, ping information, etc...

StartScanning

Sent to server, to start scanning for devices across supported busses.

StopAllDevices

Sent to server, stops actions of all currently connected devices.

StopCmd

Sent to server, stops actions of all devices, a specific device, or a specific feature.

StopDeviceCmd

Sent to server, stops actions of a specific device.

StopScanning

Sent to server, to stop scanning for devices across supported busses.
